# AI Personality Builder - Prototype Presentation Notes

## What it is

My project is called **AI Personality Builder**. It is an interactive p5.js prototype that looks like an official software tool for configuring the personality of an AI robot.

The user goes through a guided setup, answers 20 short either-or questions, and then the system compiles a personality package and installs it into the robot.

It looks clean, polished, and trustworthy on purpose, almost like a real product.

## How it connects to the final project prompt

For the final project, we were asked to imagine y/our pluriverse through new media and think about what kind of future we want technology to help create.

My project approaches that by showing the opposite: a future where software feels neutral and efficient, but quietly reduces identity into data and fixed categories.

So instead of presenting a perfect future directly, I am using a dystopian interface as a critical response. It shows the kind of system I do **not** want to normalize, which helps me think about what kind of pluriversal future I actually value.

## My vision for y/our pluriverse

The world I am imagining is one where technology does not flatten people into profiles, scores, or personality packages.

I am interested in a future where systems support agency, respect complexity, and leave room for many different ways of being. In that sense, my project is critical because it shows a system that fails to do that.

It treats the user as just **Human** and puts more value on building the robot than understanding the person. That imbalance is intentional.

## Themes and patterns explored

The project mainly connects to these course patterns:

- **agency**: the user can only choose from limited options, while the system decides what those choices mean
- **voice**: the interface controls how the user is allowed to express themselves
- **privacy**: the system collects personal preference data and turns it into a final profile
- **engagement and attention**: the quiz format is familiar and easy to follow, which helps the system gain trust

More broadly, the project explores:

- profiling and categorization
- authority and trust in software
- identity being reduced into data
- the tension between convenience and control

## Inspirations

The main inspiration came from MBTI-style personality quizzes, online personality tests, and systems that sort people into categories while sounding confident and scientific.

I was also inspired by official-looking software design, especially interfaces that feel calm, professional, and believable right away.

Another important influence is dystopian design. The software keeps calling the user **Human**, which is intentional. It makes the system feel detached, like it cares more about configuring the robot than about the person using it.

## What I am trying to achieve

I am trying to create an experience that works on two levels.

On one level, it feels like a believable futuristic tool for building a robot personality.

On another level, it becomes a critique of how digital systems classify people, simplify identity, and present those judgments as if they are objective.

So I am basically trying to make a prototype that feels polished and convincing, while also making the viewer question the logic behind that kind of system.

## How the prototype works

The prototype starts with a landing screen and consent step. After that, the user moves through 20 scenario-based questions. Each question gives two possible behaviors for the AI robot.

Behind the scenes, the answers still map to MBTI-style personality axes, but the final result does not show the four-letter code directly. Instead, it gives a written personality description.

After the results, the user can choose to install the personality. The system then shows a transfer screen and a final installation complete screen, as if the robot has now received that personality package.

## Process and methods

I made the project as a browser-based interactive prototype using **p5.js**, along with HTML, CSS, and JSON.

The main process was:

- researching MBTI, 16Personalities, and critiques of personality testing
- writing scenario-based questions that still map to MBTI-style axes
- designing the interface to feel official, polished, and slightly dystopian
- building the full user flow from consent screen to installation complete screen

I chose this medium because the message depends on the interface itself. The project would not communicate the same idea as clearly if it were only a static image or text, because the feeling of moving through a guided software system is part of the critique.

## What is still left to do

This is still a prototype, so the main things I want to add next are:

- a feature that asks for the robot's name at the beginning
- using that robot name throughout the full experience
- click sounds across the interface
- a heartbeat sound on the results page, so it feels like the robot is "alive" once the personality is installed
- a final polish and testing pass

## Short closing

Overall, **AI Personality Builder** is a prototype that looks like a helpful future tool, but is really about profiling, control, and the authority of digital systems. It uses a familiar quiz format to ask bigger questions about how identity can be shaped, reduced, and packaged by software, and what kind of technological future we actually want to build instead.
