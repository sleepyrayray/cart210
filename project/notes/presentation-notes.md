# AI Personality Builder - Prototype Presentation Notes

## Short project description

**AI Personality Builder** is an interactive prototype that looks like official software for giving an AI robot a personality. The user answers 20 short questions, the system generates a personality profile, and then it "installs" that personality into the robot.

## What it is

My project is called **AI Personality Builder**. It is an interactive p5.js prototype that looks like an official software tool for configuring the personality of an AI robot.

The user goes through a guided setup, answers 20 short questions, and then the system compiles and installs a personality package into the robot.

## Connection to the prompt

My project responds to the concept of pluriverse by showing the opposite idea: a future where software feels helpful and efficient, but quietly reduces identity into data and fixed categories.

So instead of showing an ideal future directly, I am using a dystopian interface to question the kind of technological future I do not want to normalize.

## Vision, themes, and patterns

The kind of future I value is one where technology supports complexity, agency, and different ways of being.

This project mainly connects to:

- **agency**: the user can only choose from limited options, while the system decides what those choices mean
- **voice**: the interface controls how the user is allowed to express themselves
- **privacy**: the system collects preference data and turns it into a final profile
- **engagement and attention**: the quiz format feels familiar, which helps the system gain trust

It also explores profiling, categorization, and authority in software.

## Inspirations

My main inspirations were MBTI-style personality quizzes, online personality tests, and official-looking software interfaces that feel calm, polished, and legit.

Another important part of the concept is that the system calls the user **Human**. That wording is intentional. It makes the software feel detached, like it cares more about configuring the robot than about the person using it.

## What I am trying to achieve

I want the piece to work on two levels.

At first, it feels like a legitimate futuristic tool for building a robot personality.

But underneath that, it becomes a critique of how digital systems classify people, simplify identity, and present those judgments as if they are objective.

## How it works

The prototype starts with a landing and consent screen. After that, the user moves through 20 scenario-based questions, each with two possible behaviors for the AI robot.

Behind the scenes, the answers still map to MBTI-style personality axes, but the result does not show the four-letter code directly. Instead, it gives a written personality description.

Then the user can choose to install that personality that's been generated, and the system finishes with a transfer screen and an installation complete screen.

## Process and method

I made the project as a browser-based interactive prototype using **p5.js**, HTML, CSS, and JSON.

The process included researching MBTI and critiques of personality testing, writing the set of questions, and designing the interface to feel legit, official, and somewhat dystopian.

I chose this medium because the message depends on the interface itself. Moving through the software is part of the critique.

## What is still left

The main things I still want to add are:

- a robot naming feature at the beginning
- using that name throughout the experience
- click sounds across the interface
- a heartbeat sound on the results page (to make the user feel like they had created life through generating a personality; as if they're giving life to their AI robot)
- a final polish and testing pass

## Short closing

Overall, **AI Personality Builder** looks like a helpful futuristic app, but it is really about profiling, control, and the authority of digital systems. It uses a familiar quiz format to ask what kind of technological future we actually want to build instead.
