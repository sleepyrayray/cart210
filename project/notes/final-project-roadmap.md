## Step-by-step plan

### 1. Lock the prototype
Goal:
- treat the current build as the finished creative response

Do:
- do one last quick QA pass
- only fix obvious bugs or broken layout issues
- capture the final project link, screenshots, and any images needed for slides or submission

### 2. Gather the bibliography sources
Goal:
- build the source base before writing the statement and bibliography

Do:
- choose at least 5 total sources
- make sure at least 4 are from class readings
- keep the full citation info in one place right away
- choose sources that speak to profiling, interface power, digital categorization, AI futures, and pluriversal thinking

### 3. Draft the artist statement
Goal:
- write a clear 250-300 word statement that explains the project simply

Make sure it covers:
- how the project responds to the prompt
- your vision for your part of the pluriverse
- which ideas influenced the project
- how those ideas appear in the work
- how you made it and why the medium matters

### 4. Build the annotated bibliography
Goal:
- turn the source list into the required bibliography format

Do:
- pick MLA, APA, or Chicago and stay consistent
- write around 3 sentences per source
- explain how each source shaped the concept, writing, interface, or critique
- include artworks, films, or references outside the course if they genuinely helped

### 5. Shape the presentation
Goal:
- turn the project into a clear 5-10 minute talk

Suggested slide flow:
1. title + short description
2. what the project is
3. connection to the course prompt and pluriverse
4. themes and patterns explored
5. process and tools used
6. short demo / screenshots of the prototype
7. what the project is trying to critique or achieve
8. closing reflection

### 6. Package everything for submission
Goal:
- make the final submission easy to open and easy to grade

Do:
- use a PDF or a doc/PDF with the project link on the first page
- include screenshots if the grader needs visual context
- include the artist statement and annotated bibliography in the same package or in clearly labeled companion files
- avoid Apple-only formats

### 7. Final check and submit
Goal:
- submit cleanly before the real deadline

Do:
- check the link one more time
- make sure all files open correctly
- confirm the artist statement length
- confirm the bibliography has enough sources
- submit before April 18, 2026 at 11:59 PM
